# Security

If you believe you have found a security vulnerability in RAAS, please report it responsibly.

**Please do not** open a public GitHub issue for undisclosed security bugs.

Instead, contact the maintainer privately (see author email in `raas.py --version` or the repository profile). Include:

- A short description of the issue and its impact
- Steps to reproduce (or proof-of-concept) if possible
- Affected versions or commit, if known

We aim to acknowledge reports within a few days and coordinate a fix and disclosure timeline with you.

For dependency advisories, keep `pip` and `requirements.txt` pins updated and follow your distribution’s security updates for system packages (OpenSSH, systemd, etc.) used alongside RAAS.
